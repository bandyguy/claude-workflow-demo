When working on issues, first make sure you are working off of an up-to-date `main` branch. Create a branch with naming convention `issue-<issue number>-issue-description  Then determine path forward. If there are multiple reasonable paths, propose those paths along with pros and cons. Get approval from user before moving forward with implementation. Once complete (and with user's approval) make a commit, push it up, and use `gh` to create a pull request for the branch.
